# 🤖 AI Hardware PM & Engineer Agent

> 一个基于大语言模型（LLM）的自动化智能硬件设计流工具。从“天马行空的想法”到“严谨的硬件物料清单（BOM）与接线图”，只需要一次对话。

## 💡 项目简介

本项目旨在大幅降低智能硬件的设计门槛。通过构建双 Agent 架构（PM Agent + Hardware Agent），系统能够自动与用户进行启发式对话，收敛模糊需求，并最终输出可执行的底层工程方案。

无论是为了参加 **全国大学生电子设计竞赛（电赛）** 快速构建初期方案，还是为了在 **嘉立创EDA** 中进行后续的 PCB Layout 与打样验证，本项目都能帮你迅速跨越“从 0 到 1”的需求鸿沟。

## ✨ 核心特性

- **👔 PM Agent (产品经理模式)**：模拟资深硬件产品经理，通过多轮对话引导用户，提取核心工程参数（供电、尺寸、交互、联网需求），并自动生成标准化 JSON 规格书（PRD）。
- **🛠️ Hardware Agent (硬件工程师模式)**：接收 PRD 后，严格基于市面常见、成熟的电子模块进行选型，规避复杂的底层原理图设计，直接输出 **小白友好** 的 BOM 表和傻瓜式接线图。
- **⚙️ 自动化流水线 (Pipeline)**：支持从需求沟通到方案输出的全自动无缝交接，一键运行。
- **🔌 真实物料映射**：内置本地数据库查询逻辑（模拟立创商城数据），方案生成严格挂钩真实元器件编号（如 **C编号**）与引脚定义，绝不捏造。

## 📂 项目结构与代码实现

本项目由以下核心模块组成：

| 文件名 | 职能描述 | 核心逻辑 |
| :--- | :--- | :--- |
| `pipeline.py` | **总控流水线** | 串联 PM 与硬件工程师，实现从对话到 JSON 解析再到方案生成的闭环。 |
| `PM_Agent.py` | **需求分析师** | 注入 PM 灵魂，负责将模糊想法转化为包含 `Core_Modules` 的结构化 PRD。 |
| `Hardware_Agent.py` | **底层架构师** | 负责物料提取、本地查库（C编号匹配）及最终工程方案生成。 |
| `PM_Requirement_Spec.json` | **数据中转站** | 存储由 AI 自动生成的标准化硬件需求规格书。 |

## 🚀 快速开始

### 1. 环境准备
确保你的电脑上已安装 Python 3.8+，并安装 Google GenAI SDK：
```bash
pip install google-genai
```
2. 配置 API Key 与网络代理
为了确保在国内环境下稳定调用 Gemini API，请在代码中配置您的 API 密钥及代理端口（脚本默认使用 127.0.0.1:7890）：
# 在脚本中修改以下配置
```bash
API_KEY = "YOUR_GEMINI_API_KEY"
os.environ['HTTP_PROXY'] = '[http://127.0.0.1:7890](http://127.0.0.1:7890)'
os.environ['HTTPS_PROXY'] = '[http://127.0.0.1:7890](http://127.0.0.1:7890)'
```
3. 运行项目
```bash
python pipeline.py
```

注意事项
本项目强依赖于 Google Gemini API，请确保你的 API Key 额度充足且网络环境畅通。
生成的 BOM 和接线图为 AI 基于经验输出，实际打样制板前（尤其涉及 220V 强电或大电流场景），请务必进行人工核对与专业工程验证，注意用电安全。
